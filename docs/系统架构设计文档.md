## 1. 项目概述

### 1.1 项目简介

实验室预约与设备管理系统（LabManagementSystem）是一个面向高校或科研机构的综合性管理平台，旨在实现实验室资源（实验室、设备）的数字化管理，以及预约流程的自动化处理。系统采用现代化的前后端分离架构，提供直观易用的 Web 界面，支持多角色协同操作。

### 1.2 系统目标

- **资源管理**：集中管理实验室、设备等物理资源的信息
- **预约服务**：提供便捷的在线预约功能，支持多种预约场景
- **流程审批**：支持预约审批流程，确保资源使用的合理性
- **统计分析**：提供数据统计和可视化功能，辅助决策
- **审计追溯**：记录关键操作日志，支持问题追溯

### 1.3 用户角色

| 角色 | 权限描述 |
|------|----------|
| **管理员 (ADMIN)** | 最高权限，管理所有资源、用户、审批规则，可查看系统审计日志 |
| **教师 (TEACHER)** | 管理个人课程、审批学生预约申请、管理所负责实验室 |
| **学生 (STUDENT)** | 浏览资源、提交预约申请、查看个人预约记录 |

## 2. 整体架构

### 2.1 架构模式

本系统采用**前后端分离（Separation of Frontend and Backend）**的架构模式，这是一种现代 Web 应用开发的主流架构方式。

```
┌─────────────────────────────────────────────────────────────────────┐
│                         客户端层 (Client Layer)                       │
│                  ┌─────────────────────────────┐                    │
│                  │      Vue 3 SPA 应用          │                    │
│                  │   (浏览器 / 移动端 Web)      │                    │
│                  └─────────────────────────────┘                    │
└─────────────────────────────────────────────────────────────────────┘
                                  │
                    HTTP/HTTPS (JSON)
                                  │
┌─────────────────────────────────────────────────────────────────────┐
│                         应用层 (Application Layer)                    │
│                  ┌─────────────────────────────┐                    │
│                  │     Spring Boot 后端服务     │                    │
│                  │        (REST API)           │                    │
│                  └─────────────────────────────┘                    │
└─────────────────────────────────────────────────────────────────────┘
                                  │
                          TCP/IP
                                  │
┌─────────────────────────────────────────────────────────────────────┐
│                         数据层 (Data Layer)                          │
│                  ┌─────────────────────────────┐                    │
│                  │        MySQL 8.x            │                    │
│                  │     (关系型数据库)           │                    │
│                  └─────────────────────────────┘                    │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 架构特点

1. **职责分离**：前端负责用户交互和界面渲染，后端负责业务逻辑和数据处理
2. **技术异构**：前端和后端可以使用各自最适合的技术栈
3. **独立部署**：前后端可以独立开发和部署，互不影响
4. **接口标准化**：通过 RESTful API 实现标准化通信
5. **可扩展性**：支持水平扩展，应对高并发场景

### 4.3 核心模块说明

#### 4.3.1 统一响应机制

系统采用统一的响应格式，便于前端处理和错误追踪：

```java
// ApiResponse.java
public class ApiResponse<T> {
    private int code;      // 状态码
    private String message; // 消息
    private T data;        // 数据
    private long timestamp; // 时间戳
    private String requestId; // 请求 ID（用于追踪）
}

// PageResponse.java (分页响应)
public class PageResponse<T> {
    private List<T> items;   // 数据列表
    private long total;      // 总数
    private int page;        // 当前页
    private int pageSize;    // 每页大小
    private int totalPages;  // 总页数
}
```

```java
// BusinessException.java - 自定义业务异常
public class BusinessException extends RuntimeException {
    private final ErrorCode errorCode;

// GlobalExceptionHandler.java - 全局异常处理器
@RestControllerAdvice
public class GlobalExceptionHandler {
    @ExceptionHandler(BusinessException.class)
    public ApiResponse<Void> handleBusinessException(BusinessException e) {
        // 记录错误日志
        // 返回统一错误响应
    }

```

#### 4.3.3 安全配置

```java
// SecurityConfig.java
@Configuration
@EnableWebSecurity
@EnableMethodSecurity(prePostEnabled = true)
public class SecurityConfig {
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) {
        http
            .csrf(csrf -> csrf.disable())
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/auth/**").permitAll()
                .requestMatchers("/api/admin/**").hasRole("ADMIN")
                .requestMatchers("/api/teacher/**").hasAnyRole("ADMIN", "TEACHER")
                .anyRequest().authenticated()
            )
            .oauth2ResourceServer(oauth2 -> oauth2
                .jwt(jwt -> jwt.jwtAuthenticationConverter(jwtAuthenticationConverter()))
            );
    }
}
```

#### 5.3.1 Axios 请求封装

```typescript
// request.ts
import axios, { AxiosRequestConfig, AxiosResponse } from 'axios'
import { useUserStore } from '@/store/user'
import router from '@/router'

const request = axios.create({
  baseURL: '/api',
  timeout: 10000,
})

// 请求拦截器
request.interceptors.request.use(
  (config) => {
    const userStore = useUserStore()
    if (userStore.token) {
      config.headers.Authorization = `Bearer ${userStore.token}`
    }
    return config
  },
  (error) => Promise.reject(error)
)

// 响应拦截器
request.interceptors.response.use(
  (response: AxiosResponse) => {
    const { code, data, message } = response.data
    if (code === 200) {
      return data
    }
    return Promise.reject(new Error(message || '请求失败'))
  },
  (error) => {
    if (error.response?.status === 401) {
      // token 过期或无效，清除状态并跳转登录
      const userStore = useUserStore()
      userStore.logout()
      router.push('/login')
    }
    return Promise.reject(error)
  }
)

export default request
```

#### 5.3.2 Pinia 用户状态管理

```typescript
// store/user.ts
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import { login, logout } from '@/api/auth'
import router from '@/router'

export const useUserStore = defineStore('user', () => {
  // 状态
  const token = ref(localStorage.getItem('token') || '')
  const userInfo = ref(JSON.parse(localStorage.getItem('userInfo') || 'null'))

  // 计算属性
  const isLoggedIn = computed(() => !!token.value)
  const role = computed(() => userInfo.value?.role || '')
  const isAdmin = computed(() => role.value === 'ADMIN')
  const isTeacher = computed(() => role.value === 'TEACHER')
  const isStudent = computed(() => role.value === 'STUDENT')

  // 方法
  async function doLogin(username: string, password: string) {
    const response = await login({ username, password })
    token.value = response.token
    userInfo.value = response.user
    localStorage.setItem('token', token.value)
    localStorage.setItem('userInfo', JSON.stringify(userInfo.value))
  }

  function logout() {
    token.value = ''
    userInfo.value = null
    localStorage.removeItem('token')
    localStorage.removeItem('userInfo')
    router.push('/login')
  }

  return {
    token,
    userInfo,
    isLoggedIn,
    role,
    isAdmin,
    isTeacher,
    isStudent,
    doLogin,
    logout
  }
})
```

## 8. 安全机制

### 8.1 认证体系

系统采用 **JWT (JSON Web Token)** 实现无状态认证：

```java
// JWT 流程
1. 用户提交用户名密码
2. 后端验证成功后生成 JWT
3. 前端存储 JWT（localStorage）
4. 后续请求携带 JWT
5. 后端验证 JWT 并提取用户信息
6. 基于用户角色进行权限控制
```

**Token 结构：**
```
Header: {"alg": "HS256", "typ": "JWT"}
Payload: {
  "sub": "user_id",
  "username": "zhangsan",
  "role": "STUDENT",
  "iat": 1704278400,
  "exp": 1704282000
}
Signature: HMAC-SHA256(secret, base64UrlEncode(header) + "." + base64UrlEncode(payload))
```

### 8.2 权限控制

采用 **RBAC (Role-Based Access Control)** 模型：

| 角色 | 权限 |
|------|------|
| **ADMIN** | 管理系统所有资源、用户、配置 |
| **TEACHER** | 管理课程、审批预约、管理实验室 |
| **STUDENT** | 提交预约、查看资源、查看通知 |

**权限配置示例：**
```java
// 方法级安全控制
@PreAuthorize("hasRole('ADMIN')")
@GetMapping("/users")
public ApiResponse<List<UserResponse>> getAllUsers() { ... }

@PreAuthorize("hasAnyRole('ADMIN', 'TEACHER')")
@PostMapping("/reservations")
public ApiResponse<ReservationResponse> createReservation(...) { ... }
```

### 8.3 安全措施

| 措施 | 说明 |
|------|------|
| **密码加密** | 使用 BCrypt 或 Argon2 加密存储 |
| **Token 时效** | Token 有效期 1 小时，支持刷新 |
| **HTTPS** | 生产环境强制 HTTPS |
| **CORS** | 跨域资源共享配置 |
| **请求 ID** | 每个请求分配唯一 ID，便于追踪 |
| **输入校验** | 使用 Spring Validation 校验输入 |
| **SQL 注入防护** | 使用预编译语句（MyBatis #{}） |
| **XSS 防护** | 前端转义输出 |
| **敏感日志脱敏** | 日志中不记录密码等敏感信息 |

### 8.4 审计日志

系统记录所有关键操作的审计日志：

| 审计内容 | 说明 |

| 操作 IP | 客户端 IP 地址 |
| 资源信息 | 操作对象类型和 ID |
| 变更内容 | 修改前的值和修改后的值 |

### 10.2 冲突检测算法

```java
// 预约冲突检测逻辑
public boolean checkConflict(Long labId, LocalDateTime startTime, LocalDateTime endTime) {
    // 查询同一实验室在时间范围内已有预约
    List<Reservation> existingReservations = reservationRepository
        .findByLabIdAndTimeRange(labId, startTime, endTime);

    // 检查是否有非取消状态的预约
    return existingReservations.stream()
        .anyMatch(r -> !r.getStatus().equals(ReservationStatus.CANCELLED));
}

// 冲突类型判断
public ConflictType checkConflictType(Long labId, LocalDateTime startTime, LocalDateTime endTime) {
    // 完全重叠
    if (hasCompleteOverlap(labId, startTime, endTime)) {
        return ConflictType.COMPLETE_OVERLAP;
    }
    // 部分重叠
    if (hasPartialOverlap(labId, startTime, endTime)) {
        return ConflictType.PARTIAL_OVERLAP;
    }
    // 包含关系
    if (hasContainment(labId, startTime, endTime)) {
        return ConflictType.CONTAINMENT;
    }
    return ConflictType.NONE;
}
```
